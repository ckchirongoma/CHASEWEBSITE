Accessibility & UX Best Practices
â€¢ Semantic HTML & ARIA: Use <header>, <main>, <footer>, <nav>, <section>, <article>,

â€¢ Alt Text & Images: Every image must have meaningful alt text or an empty alt (alt="") if

â€¢ Keyboard & Screen Reader: Ensure all interactive elements (links, buttons, form fields, chat

â€¢ ARIA Roles: Only use ARIA roles/states when necessary. For example, use role="status" for

â€¢ Responsive Design: The layout must adapt to mobile/tablet. Navigation becomes a hamburger

â€¢ Microinteractions:

â€¢ Transitions: Use smooth 150â€“300ms transitions for hover and focus states. Use eased

â€¢ Reduced Motion: Respect prefers-reduced-motion: if set, disable non-essential

â€¢ Forms: Form inputs should have clear placeholders or labels, visible error messages, and

â€¢ Typography Hierarchy: Headings must descend logically (H1 only once, followed by H2,
